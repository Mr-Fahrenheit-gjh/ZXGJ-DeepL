# A股 T+0 算法构建和优化执行计划

**标的：中芯国际 688981.SH；平台：VeighNa / vn.py；周期：5分钟及以上 K 线**

> 先明确边界：A股股票通常不是严格意义上的当日买入、当日卖出 T+0。对 688981 这类科创板股票，实际项目应设计为**“底仓回转 / 库存 T+0”**：用已有持仓做日内高抛低吸，避免违反 T+1 约束。上交所资料显示，中芯国际代码为 688981，属于上交所科创板标的；科创板交易规则需按上交所相关制度执行。([ SSE][1])
> VeighNa 是 Python 开源量化交易平台，提供 CTA 策略、回测、数据管理、风险管理、算法交易等模块，可作为本项目的交易、回测和实盘框架基础。([vnpy.com][2])

## 3. 阶段一：需求定义与交易规则设计

### 3.1 时间

**第 1 周**

### 3.2 关键任务

1. 明确交易模式

   * 不做违规“裸 T+0”。
   * 采用**底仓 T+0**：

     * 账户持有一定数量 688981 底仓。
     * 日内先卖后买，或先买后卖但卖出部分不得超过可用底仓。
     * 每日收盘前尽量恢复目标底仓。

2. 明确预测目标
   推荐设置三类标签：

   | 标签类型 | 定义                 | 用途   |
   | ---- | ------------------ | ---- |
   | 二分类  | 未来 N 根 K 线收益率 > 阈值 | 方向预测 |
   | 三分类  | 上涨 / 震荡 / 下跌       | 信号过滤 |
   | 回归   | 未来 N 根 K 线收益率      | 仓位强度 |

3. 明确预测周期

   * 主周期：5分钟。
   * 辅助周期：15分钟、30分钟、60分钟。
   * 预测窗口：

     * 5分钟预测未来 1、3、6 根 K 线。
     * 即未来 5、15、30 分钟收益率。

4. 明确交易限制

   * 单日最大交易次数。
   * 单笔最大仓位。
   * 最大日内亏损。
   * 最大回撤停机线。
   * 交易时间过滤：开盘前 10 分钟、收盘前 10 分钟单独处理。

## 4. 阶段二：数据构建

### 4.1 时间

**第 1–2 周**

### 4.2 数据范围

| 数据类型  | 内容                          |
| ----- | --------------------------- |
| K 线数据 | 1分钟、5分钟、15分钟、30分钟、60分钟、日线   |
| 成交数据  | 成交量、成交额、VWAP、换手率            |
| 盘口数据  | 如可获得，加入买卖一档至五档深度            |
| 市场数据  | 上证指数、科创50、半导体指数、北向资金、行业 ETF |
| 事件数据  | 财报、公告、停复牌、重大新闻，初期可不纳入模型     |
| 交易成本  | 佣金 0.1%、滑点 0.05%            |

### 4.3 数据处理流程

1. **行情清洗**

   * 去除停牌、异常价格、成交量为零的 K 线。
   * 处理缺失值。
   * 校验开高低收逻辑：

     * `high >= max(open, close)`
     * `low <= min(open, close)`

2. **周期聚合**

   * 从 1分钟数据聚合为 5m / 15m / 30m / 60m。
   * 保证不同周期特征不产生未来函数。

3. **复权处理**

   * 采用前复权或后复权。
   * 回测时价格和成交逻辑必须保持一致。

4. **样本对齐**

   * 所有特征只能使用当前时点及以前数据。
   * 标签使用未来收益，严格向后偏移。

### 4.4 标签设计

建议初期使用二分类标签：

```text
future_return = close[t + horizon] / close[t] - 1

label = 1, if future_return > transaction_cost + threshold
label = 0, otherwise
```

其中：

```text
transaction_cost = commission + slippage
                 = 0.10% + 0.05%
                 = 0.15%
```

建议阈值：

| 预测周期 | 初始阈值        |
| ---- | ----------- |
| 5分钟  | 0.15%–0.25% |
| 15分钟 | 0.25%–0.40% |
| 30分钟 | 0.40%–0.60% |

## 5. 阶段三：特征工程

### 5.1 时间

**第 2–3 周**

### 5.2 技术指标特征

| 类型   | 特征                            |
| ---- | ----------------------------- |
| 趋势   | MA、EMA、MACD、ADX、DMI           |
| 动量   | RSI、ROC、Momentum、CCI          |
| 波动率  | ATR、布林带宽度、历史波动率、Parkinson 波动率 |
| 成交量  | OBV、MFI、成交量均线、量价背离            |
| 价格结构 | K线实体、上下影线、收盘价位置、跳空幅度          |
| 均值回复 | Z-score、布林带偏离度、VWAP 偏离        |
| 市场状态 | 指数收益、行业指数收益、相对强弱              |

### 5.3 时序特征

构建滚动窗口：

```text
lookback_window = 32 / 64 / 128 根 K 线
input_shape = [batch_size, lookback_window, feature_dim]
```

建议特征：

* 滚动均值
* 滚动标准差
* 滚动偏度
* 滚动峰度
* 滚动最大回撤
* 滚动收益分位数
* 过去 N 根 K 线涨跌比例
* 过去 N 根 K 线连续上涨 / 下跌次数

### 5.4 特征选择

采用三层筛选：

1. **基础过滤**

   * 删除缺失率高于 20% 的特征。
   * 删除方差过低特征。
   * 删除高度共线特征，相关系数阈值可设为 0.95。

2. **模型过滤**

   * LightGBM / XGBoost 重要性。
   * 随机森林重要性。
   * Logistic L1 正则筛选。

3. **稳定性过滤**

   * 按年份 / 月份计算特征重要性。
   * 删除只在单一时期有效的特征。
   * 保留跨市场状态稳定的特征。

## 6. 阶段四：模型构建

### 6.1 时间

**第 3–5 周**

### 6.2 主模型：Transformer-LSTM

推荐结构：

```text
Input Sequence
   ↓
Linear Embedding
   ↓
Positional Encoding
   ↓
Transformer Encoder
   ↓
LSTM Layer
   ↓
BatchNorm / LayerNorm
   ↓
Dropout
   ↓
MLP Head
   ↓
Sigmoid Output
```

### 6.3 模型设计要点

| 组件                    | 作用                    |
| --------------------- | --------------------- |
| Transformer Encoder   | 捕捉长距离依赖和不同时间点之间的注意力关系 |
| LSTM                  | 捕捉局部时间序列动态            |
| Residual Connection   | 缓解深层网络退化              |
| BatchNorm / LayerNorm | 提高训练稳定性               |
| Dropout               | 降低过拟合                 |
| Xavier 初始化            | 用于 Linear / MLP 层     |
| Orthogonal 初始化        | 用于 LSTM 权重            |

### 6.4 对照模型

至少构建四类模型：

| 模型                 | 用途       |
| ------------------ | -------- |
| 普通 LSTM            | 时间序列基线模型 |
| CNN-1D             | 捕捉局部形态结构 |
| MLP                | 静态特征基线   |
| LightGBM / XGBoost | 表格型强基线   |
| Transformer-LSTM   | 主模型      |
| Ensemble           | 最终交易信号模型 |

### 6.5 集成方法

初期建议采用概率加权融合：

```text
p_final = w1 * p_transformer_lstm
        + w2 * p_lstm
        + w3 * p_cnn
        + w4 * p_mlp
        + w5 * p_lgbm
```

权重通过验证集 AUC、F1、收益表现联合确定。

进阶版本采用 Stacking：

```text
Base Models → Out-of-fold Predictions → Meta Model → Final Probability
```

## 9. 阶段七：交易信号设计

### 9.1 时间

**第 7 周**

### 9.2 信号转换逻辑

模型输出为上涨概率：

```text
p = model.predict_proba(x)
```

信号规则：

| 条件                | 动作      |
| ----------------- | ------- |
| p > 0.60 且趋势过滤通过  | 买入 / 回补 |
| p < 0.40 且持有可卖底仓  | 卖出 / 减仓 |
| 0.40 <= p <= 0.60 | 观望      |
| 波动率异常             | 降低仓位    |
| 当日亏损触发阈值          | 停止交易    |

### 9.3 趋势过滤器

避免模型在噪声区频繁交易：

```text
交易条件 = 模型信号 + 趋势过滤 + 成交量过滤 + 波动率过滤
```

示例：

* 价格高于 VWAP，优先做多回补。
* 价格低于 VWAP，优先高抛或减仓。
* ATR 过高时降低仓位。
* 成交量低于均值时不交易。
* 开盘前 10 分钟只观察，不主动交易。

### 9.4 仓位函数

建议用概率强度控制仓位：

```text
target_position_ratio = min(max_position, k * abs(p - 0.5))
```

示例：

| 概率      | 仓位       |
| ------- | -------- |
| 0.55    | 轻仓       |
| 0.60    | 中仓       |
| 0.70    | 重仓       |
| 0.80 以上 | 仍受最大仓位限制 |

## 10. 阶段八：回测系统与风险管理

### 10.1 时间

**第 7–8 周**

### 10.2 回测模式

必须采用滚动窗口回测：

```text
训练窗口 → 验证窗口 → 测试窗口 → 前移窗口 → 重复
```

避免一次性全样本训练导致过拟合。

### 10.3 成本模型

```text
commission = 0.10%
slippage = 0.05%
total_cost = 0.15%
```

每次买卖均计入成本。

### 10.4 滑点模型

初期：

```text
buy_price = close * (1 + 0.0005)
sell_price = close * (1 - 0.0005)
```

进阶：

* 根据成交量比例动态调整滑点。
* 根据波动率动态调整滑点。
* 根据买卖盘口深度估计冲击成本。

### 10.5 风控规则

| 风控项       | 建议参数         |
| --------- | ------------ |
| 单笔最大仓位    | 总资产 10%–20%  |
| 单日最大交易次数  | 5–20 次       |
| 单日最大亏损    | 1%–2%        |
| 单笔止损      | 0.5%–1.5%    |
| 最大回撤停机    | 5%–10%       |
| 收盘前强制恢复底仓 | 14:45 后执行    |
| 连续亏损暂停    | 连续 3–5 笔亏损暂停 |

### 10.6 绩效指标

必须输出：

```text

最大回撤
```

## 12. 阶段十：仿真交易与上线验证

### 12.1 时间

### 12.2 仿真交易阶段

至少运行 **20–40 个交易日**。

* 信号是否稳定。
* 交易频率是否过高。
* 滑点是否明显偏离假设。

## 14. 关键技术风险与应对

| 风险     | 表现            | 应对                                          |
| ------ | ------------- | ------------------------------------------- |
| 未来函数   | 回测很好，实盘失效     | 严格时间对齐，标签后移，特征只用历史数据                        |
| 过拟合    | 验证集好，测试集差     | Walk-forward、Dropout、Early Stopping、特征稳定性过滤 |
| 成本吞噬收益 | 毛收益为正，净收益为负   | 提高信号阈值，降低交易频率，优化滑点模型                        |
| 样本不平衡  | 上涨样本和下跌样本比例失衡 | 加权 BCE、Focal Loss、重采样                       |
| 模型漂移   | 市场风格变化后失效     | 月度再训练、漂移监控、动态模型权重                           |
| T+1 约束 | 信号要求卖出不可卖股票   | 使用可用底仓约束，不允许超出可卖数量                          |
| 流动性冲击  | 实盘成交价格差       | 限制订单规模，使用算法委托，避开低流动性时段                      |
| 黑天鹅事件  | 突发跳水或停牌       | 最大亏损停机、单日熔断、人工干预                            |

## 18. 最终执行顺序建议

优先顺序如下：

1. **先确认交易规则和 T+1 约束。**
2. **先做数据质量，不急着建复杂模型。**
3. **先做 LightGBM / LSTM 基线。**
4. **用回测收益验证标签是否有效。**
5. **确认成本后仍有收益，再做 Transformer-LSTM。**
6. **确认模型稳定后，再做 SHAP 和深度解释。**
7. **确认滚动回测有效后，再接入 VeighNa。**
8. **先仿真，不直接实盘。**
9. **小资金实盘后，再考虑仓位放大。**

这个项目的关键不在于模型复杂度，而在于：**无未来函数的数据工程、严格的 T+1/底仓约束、成本后仍有效的信号、滚动窗口验证、以及实盘风控闭环**。
