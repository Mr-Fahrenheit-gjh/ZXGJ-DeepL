# 中芯国际 688981 A 股 T+0 机器学习系统实现与运行指南

这份文档用于说明当前项目如何对应题目要求、各模块负责什么、如何一键运行实验，以及输出结果应该怎么看。

重要结论先说清楚：当前系统已经能完整跑通“数据构建 -> 特征工程 -> 双机会标签 -> 多模型训练 -> 集成信号 -> 滚动回测 -> 解释性分析 -> vn.py 事件驱动回测”的研究闭环；但这不等于已经可以投入真实资金。真实资金前还需要更长 walk-forward、影子盘、实盘前审计和人工复核。

## 1. 题目要求与代码对应关系

### 1.1 数据构建

题目要求：

```text
构建多类技术指标，进行自动化特征工程，提取时序特征，进行特征选择。
构建滚动时间窗口，创建历史序列作为模型输入。
```

对应模块：

```text
feature_engineering.py
execution_audit.py
label_builder.py
run_research_pipeline.py
```

当前实现：

- `build_basic_features()` 生成收益率、动量、K 线结构、波动率、VWAP、资金流、成交量、时间段、节假日后等特征。
- `select_feature_columns()` 自动剔除标签、未来字段、绝对价格字段、常量字段。
- `make_sequence_data()` 将二维特征表变成 `[样本数, lookback, 特征数]` 的历史序列。
- `clean_market_data_for_execution()` 会剔除不可交易/异常 K 线所在交易日，并输出清洗报告。
- `build_path_dependent_opportunity_labels()` 使用路径依赖机会标签，而不是简单固定 horizon 涨跌标签。

关键输出：

```text
feature_cols.csv
label_summary.csv
label_reason_distribution.csv
market_data_cleaning_report.json
execution_feasibility_audit.json
```

### 1.2 模型构建

题目要求：

```text
实现混合 Transformer-LSTM 或其它模型，开发多模型集成系统：
Transformer-LSTM、普通 LSTM、CNN、MLP。
使用残差连接、批量归一化、Dropout 防止过拟合。
使用 Xavier 和正交初始化优化模型权重。
```

对应模块：

```text
model_signals.py
ensemble.py
```

当前实现：

- `TransformerLSTMSequenceClassifier`
- `LSTMSequenceClassifier`
- `CNNSequenceClassifier`
- `MLPSequenceClassifier`
- `RandomForestClassifier`
- `LogisticRegression`
- `build_ensemble_signal_result()`

结构细节：

- Transformer-LSTM 使用 `TransformerEncoder` 进行自注意力建模，再接 LSTM。
- Transformer-LSTM 显式加入 `h + h0` 残差连接。
- CNN 和 MLP 使用 BatchNorm。
- 各深度模型使用 Dropout。
- Linear/Conv/Input projection 使用 Xavier 初始化。
- LSTM recurrent weights 使用 orthogonal 初始化。

### 1.3 模型训练

题目要求：

```text
实施时间序列交叉验证。
使用 Optuna 进行超参数优化。
实现早停机制，监控验证集 AUC。
采用 BCE 损失函数和 Adam 优化器。
```

对应模块：

```text
walk_forward_runner.py
validation.py
hyperparameter_optimization.py
model_signals.py
```

当前实现：

- `build_walk_forward_splits()` 做时间序列滚动切分。
- 每个 fold 内部再切 subtrain/calibration/test。
- 缩尾和标准化只 fit subtrain，避免未来函数。
- 深度模型训练使用 `BCEWithLogitsLoss`。
- 优化器使用 Adam。
- 早停监控 validation AUC。
- `run_optuna_sequence_search()` 支持对 Transformer-LSTM/LSTM/CNN/MLP 搜索学习率、dropout、hidden dim、层数等。

注意：

- 如果当前环境没有安装 `optuna`，pipeline 不会崩溃，会输出 `optuna_report.json`，状态为 `SKIPPED`。
- 安装后同一命令会自动执行 Optuna。

### 1.4 模型解释性

题目要求：

```text
集成 SHAP 分析，可视化特征对预测结果的影响。
实现梯度重要性分析，识别关键预测因子。
计算排列重要性，评估特征稳定性。
```

对应模块：

```text
explainability.py
walk_forward_runner.py
```

当前实现：

- sklearn 模型输出 permutation importance。
- sklearn 模型尝试输出 SHAP summary；若本机未安装 shap 或模型不兼容，则自动跳过。
- torch 序列模型输出 input-gradient importance。
- 每个 fold 可输出 `explainability_manifest.json`。

关键输出：

```text
fold_000/explainability/*_permutation_importance.csv
fold_000/explainability/*_gradient_feature_importance.csv
fold_000/explainability/*_gradient_time_importance.csv
fold_000/explainability/*_shap_summary.csv
fold_000/explainability/explainability_manifest.json
```

### 1.5 回测和风险管理

题目要求：

```text
开发滚动窗口回测系统，模拟真实交易环境。
集成交易成本 0.1% 佣金和 0.05% 滑点模型。
计算夏普比率、索提诺比率、最大回撤、Calmar。
实现动态仓位管理和止损机制。
使用 VeighNa 开源平台。
```

对应模块：

```text
risk_management.py
walk_forward_runner.py
vnpy_backtest.py
run_research_pipeline.py
```

当前实现：

- pandas 研究回测：`run_a_share_inventory_t0_backtest()`
- A 股底仓 T+0 状态机：
  - `sell_first`：先卖出底仓，再买回；
  - `buy_first`：先买入增强仓位，再卖出旧库存恢复底仓。
- 成本：
  - `commission_rate = 0.001`
  - `slippage_rate = 0.0005`
- 绩效指标：
  - total return
  - alpha return
  - Sharpe
  - Sortino
  - max drawdown
  - Calmar
  - win rate
  - trade count
- 风控：
  - 动态仓位
  - 止盈
  - 止损
  - 成交量参与率约束
  - 压力测试
- vn.py 事件驱动回测：
  - `build_dual_signal_inventory_t0_strategy()`
  - `run_vnpy_dual_signal_t0_backtest()`

注意：

vn.py CTA 回测本身不是 A 股券商结算系统，因此这里的 vn.py 策略是“事件驱动交叉验证”，用于验证信号时间、订单触发和成本影响，不应被单独理解为完整券商级实盘仿真。

## 3. 如何读主要输出

### 3.1 pipeline_manifest.json

记录本次实验的数据范围、清洗情况、特征数、标签口径、代码和 config 指纹。

### 3.2 walk_forward_summary.json

看整体模型和回测表现：

```text
executed_fold_count
total_trades
median_test_buy_auc
median_test_sell_auc
median_alpha_total_return
stress_min_alpha_total_return
quality_report
```

### 3.3 walk_forward_fold_summary.csv

逐 fold 看模型表现和交易表现。重点看：

```text
selected_model
test_buy_auc
test_sell_auc
alpha_total_return
alpha_max_drawdown
alpha_sharpe
alpha_sortino
alpha_calmar
trade_count
stress_min_alpha_total_return
```

### 3.4 explainability_manifest.json

说明解释性分析输出了哪些文件。

### 3.5 vnpy_backtest_report.json

说明 vn.py 是否可用、是否跑通、使用哪个 fold 的信号、事件驱动回测统计指标。

### 3.6 live_readiness_report.json

这是实盘前硬门槛文件。

```text
status = PASS
```

才允许进入人工复核和小资金仿真。

当前大多数 smoke 都会是：

```text
status = FAIL
```

这是正常的，因为 smoke 只用于验证系统，不用于证明策略可实盘。

## 5. 当前系统距离“真实资金”还差什么

现在已经更接近题目要求的完整系统，但还不是可投入资金系统。

主要缺口：

```text
1. 需要跑完整 walk-forward，而不是 quick smoke。
2. 需要安装 optuna 并完成正式超参数搜索。
3. 需要安装 shap 并确认 SHAP 输出稳定。
4. 需要比较 pandas 研究回测和 vn.py 事件回测的一致性。
5. 需要至少 20 个交易日以上影子盘。
6. 需要 live_readiness_report.json 全部硬门槛 PASS。
7. 需要人工检查交易样本，确认没有未来函数和不可成交假设。
```
