## P0：必须优先完成

### 1. Nested validation，防止自动调参过拟合

当前 `--run-wf-optuna` 会根据 walk-forward 交易结果选参数，但还没有严格区分“调参样本”和“最终评估样本”。

需要实现：

- 将数据切成 outer train / outer test；
- outer train 内部运行 Optuna 和 inner walk-forward；
- 锁定最佳参数后，只在 outer test 上评估一次；
- 输出 nested 结果文件。

建议输出：

```text
outputs/diagnostics/<run_name>/nested_validation/
  inner_optuna_trials.csv
  inner_optuna_best.json
  outer_test_summary.json
  outer_test_fold_summary.csv
  nested_validation_report.md
```

验收标准：

- 最终报告明确区分 inner tuning 和 outer evaluation；
- `outer_test_summary.json` 中的结果不能参与参数选择；
- 若 outer test alpha 明显劣化，需要标记为过拟合风险。

### 2. 滚动窗口稳定性实验

当前默认参数：

```text
train_bars = 12000 约 250 个交易日
valid_bars = 2400 约 50 个交易日
step_bars = 2400 约 50 个交易日
```

这不是乱设，但仍属于经验值。需要系统比较不同窗口。

建议实验网格：

```text
train_days: 60, 120, 180, 250, 500
test_days: 20, 40, 60
step_days: 20, 40, 60
split_mode: rolling, expanding
```

换算：

```text
bars = trading_days * 48
```

建议输出：

```text
outputs/diagnostics/<run_name>/window_stability/
  window_stability_results.csv
  window_stability_summary.json
  window_stability_report.md
```

核心指标：

- median alpha；
- mean alpha；
- stress min alpha；
- total trades；
- median buy/sell AUC；
- median drawdown；
- 不同窗口下排名是否稳定。

验收标准：

- 不能只选表现最好的窗口；
- 应选择表现稳定、交易数足够、压力测试不崩的窗口；
- 若窗口变化导致结果大幅反转，应标记为不稳定。

### 3. Label 参数稳定性实验

当前默认：

```text
horizon = 3
tp = 0.003
sl = 0.002
cost = 0.0015
```

这套标签比 fixed horizon 合理，但仍需要检验。

建议实验网格：

```text
horizon: 3, 6, 9, 12
tp: 0.002, 0.003, 0.004, 0.005
sl: 0.0015, 0.002, 0.003, 0.004
timeout_policy: final_return, negative
```

建议输出：

```text
outputs/diagnostics/<run_name>/label_stability/
  label_parameter_stability.csv
  label_model_backtest_stability.csv
  label_stability_report.md
```

核心指标：

- buy/sell 正样本比例；
- timeout / take_profit / stop_loss 分布；
- 每组 label 下的模型 AUC；
- 每组 label 下的交易 alpha；
- label 参数改变后信号是否稳定。

验收标准：

- 正样本比例不能极端稀疏；
- 标签改变后不能完全换一套结论；
- 如果某个 label 参数只在单一窗口盈利，应视为过拟合嫌疑。

### 4. 概率分桶交易价值诊断

当前结果显示 AUC 有边际，但交易亏损。必须验证模型概率排序是否真的有交易价值。

需要按 `p_buy` / `p_sell` 分桶：

```text
q50-q70
q70-q80
q80-q90
q90-q95
q95-q98
q98-q100
```

每桶输出：

- 样本数；
- label 正例率；
- 平均 buy/sell path return；
- 成本后收益；
- 触发交易次数；
- 胜率；
- 平均盈亏；
- 盈亏比。

建议输出：

```text
outputs/diagnostics/<run_name>/probability_buckets/
  buy_probability_bucket_analysis.csv
  sell_probability_bucket_analysis.csv
  bucket_trade_value_report.md
```

验收标准：

- 高分位概率桶应该比低分位桶更有交易价值；
- 若 q95/q98 桶仍无法覆盖成本，应优先修 label 或交易逻辑；
- 若高分位桶有效但整体亏，应修阈值、交易次数、仓位和退出机制。

## P1：重要优化

### 5. 特征选择自动化

当前特征工程较完整，但特征选择主要是排除泄露列、绝对量纲列和常数列，还不够强。

需要补充：

- 相关性过滤；
- 单特征 AUC / IC 筛选；
- 跨 fold permutation importance 稳定性；
- SHAP / gradient importance 稳定性；
- 特征组消融实验。

建议输出：

```text
outputs/diagnostics/<run_name>/feature_selection/
  feature_correlation_report.csv
  univariate_feature_score.csv
  fold_importance_stability.csv
  selected_features.json
  feature_selection_report.md
```

验收标准：

- 选出的特征在不同 fold 中重要性相对稳定；
- 删除低稳定性特征后，交易结果不应明显变差；
- 不能用 test fold 结果反向筛选特征。

### 6. 模型选择改成交易导向

当前 ensemble 主要按 AUC 权重。结果显示 AUC 边际不能保证交易盈利。

需要改成多指标模型选择：

```text
score = alpha + stress_alpha + AUC_edge - drawdown_penalty - turnover_penalty
```

建议：

- 保留 AUC 作为最低门槛；
- 用 calibration fold 的交易结果选择权重；
- test fold 只评估，不调权重。

输出：

```text
outputs/diagnostics/<run_name>/model_selection/
  model_selection_scores.csv
  ensemble_weights_by_fold.csv
  model_selection_report.md
```

### 7. 交易状态机细化

当前 A股底仓 T+0 状态机已经考虑底仓、可卖股、买回/卖出闭环、手续费、滑点、参与率限制、不过夜，但仍是研究级。

待优化：

- 收盘前强制恢复目标底仓；
- 午盘前后不同规则；
- 单日最大交易次数；
- 单日最大亏损；
- 连续亏损暂停；
- 多笔分层做T；
- 不同市场环境下动态止盈止损。

输出：

```text
outputs/diagnostics/<run_name>/risk_rules/
  risk_rule_comparison.csv
  risk_rule_report.md
```
